**Key Finding**: Steps 3, 4, 4b, 4c (file generation) account for 79% of elapsed time. 
All file generation steps target the AI content generation phase, which is inherently bounded
by model inference speed. Steps 1, 2, 6, 7 are fast document-read/write operations.

| Tree Size (nodes) | Check Time (s) | Linear Projection at 50 nodes |
|------------------|---------------|-------------------------------|

**Finding**: Link integrity check grows approximately linearly with node count. At 20 nodes
it is 18 s, within the 30 s budget. For very large hierarchies (>25 nodes), practitioners
should invoke `--mode incremental` to scope checks to the modified branch only.

## 4. Optimization Recommendations

### OPT-1: Use `--mode incremental` for Link-Integrity Checks in Large Hierarchies
**Target**: `hierarchy-management` §7c (Link Integrity Check)  
**Rationale**: Link-integrity check scales linearly; for hierarchies >20 nodes, full-tree
checks approach the time budget. Incremental mode scopes the check to the affected branch.  
**Expected Gain**: Reduces link-check time by ~70% for 20+ node trees.  
**Applicability**: Large multi-project hierarchies; single-developer hierarchies with <15 nodes may skip.

### OPT-2: Generate All Four Sub-Process Files in One AI Invocation
**Target**: `hierarchy-management` Steps 3, 4, 4b, 4c  
**Rationale**: Each file is currently generated as a distinct AI response. Combining all four
stubs into a single generation request reduces model-invocation round-trips from 4 to 1.  
**Expected Gain**: Reduces file-generation phase from ~8.7 s to ~4 s (est. 50% improvement).  
**Implementation Note**: SKILL.md should note practitioners may batch-request all four stubs.

### OPT-3: Reuse Parent Stereotype Annotations in Child Diagrams
**Target**: `diagram-generatecollaboration` (stereotype inference) → `hierarchy-management` §1  
**Rationale**: Child participants inherit context from the parent boundary participant. Re-inferring
stereotypes at each level adds latency. Passing `"type": "control"` annotations from parent
to child in hierarchy-metadata.json avoids redundant classification.  
**Expected Gain**: Saves ~0.5 s per decomposition; significant for deep (L4+) chains.

### OPT-4: Validate Mermaid Box Syntax Before Full Generation
**Target**: `diagram-generatecollaboration` boundary generation step  
**Rationale**: If boundary parameters are invalid (e.g., duplicate participant in two boxes),
the diagram generates but fails Mermaid rendering. Pre-validating box parameters before
generating the full sequence body avoids regeneration.  
**Expected Gain**: Eliminates regeneration penalty (~11 s) on box-rule violations; reduces
error discovery to <1 s.
